# ctfd-dynamic-values — динамические переменные для каждого участника CTFd

[🇬🇧 English version](README.md)

Позволяет задавать именованные переменные в задании, которые **детерминированно
генерируются** на основе личности участника (пользователь/команда) и
опциональной соли, используя **маску** токенов. Подставляются в описание
задания как `{{name}}` — каждый участник видит своё уникальное значение,
которое не меняется при обновлении страницы (значение — чистая функция от
входных данных, ничего случайного не хранится).

Типичное применение: уникальный IP/порт для каждого пользователя, подсеть
для команды, персональный токен встроенный прямо в текст задания.

## Установка

Положи папку в `CTFd/plugins/` под именем `ctfd-dynamic-values`:

```
CTFd/plugins/ctfd-dynamic-values/
├── __init__.py
├── generator.py
├── README.md
└── assets/
    ├── variables.js
    └── variables.css
```

В Docker стеке монтируй с этим же именем:

```yaml
- ./plugins/ctfd-dynamic-values:/opt/CTFd/CTFd/plugins/ctfd-dynamic-values:ro
```

Перезапусти CTFd. При старте плагин:
- создаёт свою таблицу (`dynamic_values`),
- регистрирует админ-скрипт добавляющий вкладку **Variables** в редактор
  задания (рядом с Files / Flags / Topics),
- патчит `read()` задания так что `{{name}}` в описании подставляется для
  каждого участника индивидуально,
- публикует `app.dynamic_values_substitute` для опциональной интеграции с флагами.

## Использование

1. Открой задание в админке, перейди во вкладку **Variables**.
2. Добавь переменную: **имя** (`target_ip`), **маска**, **область видимости**
   и опционально **соль**. Используй **Preview** чтобы увидеть пример значения.
3. Вставь `{{target_ip}}` в любом месте **описания** задания.
4. Теперь каждый участник видит своё сгенерированное значение, стабильное
   между обновлениями страницы.

### Область видимости (Scope)

| Scope     | Значение общее для… |
|-----------|---------------------|
| `user`    | каждого отдельного пользователя (по умолчанию) |
| `team`    | всех участников одной команды (если команды нет — переключается на per-user) |
| `global`  | всех участников (одно значение на всё задание) |

### Соль (Salt)

Опциональная строка добавляемая в seed. Меняй её чтобы сменить все
сгенерированные значения переменной без изменения имени или маски. Оставь
пустой если не нужно.

## Токены маски

Всё что вне токена выводится буквально. Токены потребляют детерминированную
энтропию слева направо.

| Токен            | Значение |
|------------------|----------|
| `{A-B}`          | целое число в диапазоне `[A, B]` (десятичное) |
| `{N}`            | `N` десятичных цифр |
| `{xN}` / `{XN}`  | `N` hex-символов (нижний / верхний регистр) |
| `{aN}` / `{AN}`  | `N` букв (нижний / верхний регистр) |
| `{wN}`           | `N` буквенно-цифровых символов `[a-zA-Z0-9]` |
| `\{` `\}`        | буквальные фигурные скобки |

### Примеры

| Маска                              | Пример результата |
|-------------------------------------|--------------------|
| `10.{0-255}.{0-255}.{1-254}`        | `10.137.42.200`    |
| `{1024-65535}`                      | `49213`            |
| `host-{x8}`                         | `host-3fa9c0b1`     |
| `USER-{A6}`                         | `USER-KQWZPL`        |
| `token_{w16}`                       | `token_yK1s0KqHHWq3aB9c` |
| `static-value-123` (без токенов)    | `static-value-123` (всегда) |

## Детерминизм

Seed формируется как `sha256("<scope_key>|<salt>|<challenge_id>|<name>")`,
расширяется в неограниченный поток байт для маски. Диапазоны используют
unbiased rejection sampling. Одинаковые входные данные → всегда одинаковый
результат. Поэтому значения переживают обновление страницы, и два разных
пользователя (почти всегда) получают разные значения, в то время как
`team`-scoped переменная одинакова для всех участников команды.

## Интеграция с ctfd-dynamic-flag

Если установлен **ctfd-dynamic-flag**, его флаги `dynamic_formula` могут
ссылаться на эти переменные в `source`/`secret` через тот же синтаксис
`{{name}}` — так флаг может вычисляться из того же значения которое видит
пользователь в тексте задания. Интеграция слабо связана через
`app.dynamic_values_substitute`; плагины не импортируют друг друга и каждый
работает самостоятельно.

## REST API (только админ)

| Метод | Путь | Назначение |
|--------|------|---------|
| GET    | `/api/v1/plugins/dynamic_values?challenge_id=<id>` | список переменных |
| POST   | `/api/v1/plugins/dynamic_values` | создать |
| PATCH  | `/api/v1/plugins/dynamic_values/<id>` | обновить |
| DELETE | `/api/v1/plugins/dynamic_values/<id>` | удалить |
| GET    | `/api/v1/plugins/dynamic_values/preview?mask=…&salt=…&scope=…&name=…` | предпросмотр маски |

Все эндпоинты требуют админской сессии (`@admins_only`).

## Заметки по безопасности

- Никакого выполнения кода: маски парсятся фиксированным токенайзером,
  никогда не `eval`/`exec`.
- Сгенерированные значения сами по себе не являются секретами; если флаг
  выводится из переменной, секретность обеспечивается солью/секретом флага,
  а не переменной.
- Эндпоинты записи доступны только админам, предпросмотр использует личность
  текущего админа как пример.

## Совместимость

CTFd 3.x и 4.x.
